# Diary

Session-by-session capture for project knowledge crystallization.

## Purpose

Raw experience → Crystallized learnings → Architectural decisions

**Knowledge that isn't crystallized is knowledge that will be re-learned the hard way.**

## Entry Template

```markdown
# YYYY-MM-DD — [Session Focus]

## What We Did
- [Work completed]

## What Surprised Us
- [Unexpected outcomes]

## Patterns That Emerged
- [Repeated structures in code, process, or behavior]

## Technical Decisions Made
- [And why]

## Crystallization Candidates

### → docs/learnings/
- [Learning to crystallize]

### → docs/decisions/
- [ADR to record]

### → TIP Proposal
- [If this generalizes to other projects]
```

## Flow

1. **Capture** during work
2. **Crystallize** at session end
3. **Propagate** learnings that generalize
