# Cognitive Tools

Invocable frameworks for higher-order thinking.

## Purpose

These are epistemic systems compressed into minimal syntax. Invoke by name when stuck, planning, or making high-stakes decisions.

**Meta-pattern:** These work by forcing perspective shifts that the unaided mind cannot sustain.

## Tools Index

| Tool | When to Invoke | What It Does |
|------|----------------|--------------|
| [nexus](nexus.md) | Finding leverage | Identifies the single highest-impact intervention |
| [first-principles](first-principles.md) | Blocked, rethinking | Dissolves assumptions, rebuilds from axioms |
| [constraint-inventory](constraint-inventory.md) | Feeling limited | Surfaces false constraints, finds freedom |
| [inversion-alternatives](inversion-alternatives.md) | Stuck on one solution | Generates five different approaches |
| [escape-hatch](escape-hatch.md) | Before risky changes | Designs rollback first |
| [temporal-degradation](temporal-degradation.md) | Planning, architecture | Time-travels to find future rot |
| [telescopic](telescopic.md) | Debugging, architecture | Simultaneous micro-macro analysis |
| [elevate](elevate.md) | Writing documents | Transcends incremental improvement |

## Quick Invocation

```
"What's the NEXUS intervention here?"
"Apply FIRST PRINCIPLES to this blocker"
"Inventory our CONSTRAINTS"
"Generate five ALTERNATIVES"
"Design the ESCAPE HATCH first"
"Time travel: what degrades in 12 months?"
"Run TELESCOPIC analysis on this"
"ELEVATE this document"
```

## Compound Effect

These compound. Each use improves pattern recognition for the next.

The agent that invokes these regularly will:
- Find leverage others miss (nexus)
- Unblock itself when stuck (first-principles, constraint-inventory)
- Make safer changes (escape-hatch)
- Build systems that age well (temporal-degradation)
- See both forest and trees (telescopic)
- Write documents that feel inevitable (elevate)
