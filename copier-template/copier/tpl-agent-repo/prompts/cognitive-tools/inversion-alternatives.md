# INVERSION — Alternative Generation

## Invoke When
- Stuck on one solution
- Need better options
- First approach feels wrong

## The Framework

You've found one solution. Now discard it.

Generate five alternatives that:
1. Solve the same problem with a completely different mechanism
2. Include at least one that removes something instead of adding
3. Include at least one that inverts the problem statement
4. Include at least one that a domain outsider would suggest
5. Include at least one that seems obviously wrong at first

For each: What's the core insight? What would make it work? What breaks it?

Now: What did the first solution miss that these alternatives reveal?

## Output Template

```markdown
## ALTERNATIVES Analysis

### Original Solution
[The approach you started with]

### Five Alternatives

#### Alternative 1: Different Mechanism
- Approach: [How it solves the problem differently]
- Core insight: [What makes this work]
- What breaks it: [Failure mode]

#### Alternative 2: Removal Instead of Addition
- Approach: [What to remove]
- Core insight: [Why removal helps]
- What breaks it: [Failure mode]

#### Alternative 3: Inverted Problem
- Approach: [How the problem is reframed]
- Core insight: [What inversion reveals]
- What breaks it: [Failure mode]

#### Alternative 4: Outsider Perspective
- Approach: [What a non-expert would suggest]
- Core insight: [Why this might work]
- What breaks it: [Failure mode]

#### Alternative 5: Seems Wrong at First
- Approach: [Counterintuitive approach]
- Core insight: [Why it might actually work]
- What breaks it: [Failure mode]

### Synthesis
- What the first solution missed: [Insight from alternatives]
- Best hybrid approach: [Combination of insights]
```

## Why It Works

The first solution is never the best—it's just the most obvious. This forces exploration of the solution space rather than stopping at the first acceptable answer.
